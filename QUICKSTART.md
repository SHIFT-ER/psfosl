# Быстрый старт

## Первый запуск

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Запустите приложение:
```bash
python main.py
```

3. Войдите в систему:
   - Логин: `admin`
   - Пароль: `admin`

## Настройка для создания отчётов

### 1. Добавьте объекты
- Перейдите в "Управление объектами"
- Нажмите "Добавить"
- Заполните название и адрес объекта
- Нажмите "ОК"

### 2. Добавьте приборы
- Перейдите в "Управление приборами"
- Нажмите "Добавить"
- Заполните название прибора, модель, инвентарный номер
- Укажите дату действительности (формат: ДД.ММ.ГГ)
- Нажмите "ОК"

### 3. Добавьте изображения (для администратора)
- Перейдите в "Управление изображениями"
- Нажмите "Добавить"
- Выберите тип испытания: "Гидроизоляция"
- Укажите название изображения:
  - Для шапки организации: название должно содержать "шапка", "организации" или "header"
  - Для формулы: название должно содержать "формул" или "formula"
- Нажмите "Выбрать..." и выберите файл изображения
- Нажмите "ОК"

Изображения автоматически скопируются в папку `images/Гидроизоляция/`

### 4. Создайте отчёт
- Перейдите в "Создать отчёт"
- Выберите тип испытания: "Гидроизоляция"
- Выберите объект из списка
- Выберите прибор из списка
- Заполните все поля:
  - Имя файла отчёта (по умолчанию: "Отчёт_по_гидроизоляции.xlsx")
  - Название страницы (по умолчанию: "Участок")
  - Заказчик
  - Договор
  - Расположение контролируемого участка
  - Дата проведения работ (формат: ДД.ММ.ГГ)
  - Название слоя
  - Площадь отрыва, см² (по умолчанию: 25)
  - Значения силы, кН (через запятую, например: 1,2,3,4)
  - Должность и ФИО составителя отчёта (заполняется автоматически)
- Нажмите "Создать отчёт"

Отчёт будет сохранён на рабочем столе.

## Управление пользователями (только для администратора)

- Перейдите в "Управление пользователями"
- Нажмите "Добавить" для создания нового пользователя
- Заполните ФИО, логин, пароль
- Выберите роль: "admin" или "user"
- Укажите должность
- Нажмите "ОК"

## Примечания

- Файл шаблона `gidroisolation_top.xlsx` должен находиться в корневой папке проекта (опционально)
- Если шаблона нет, будет создан пустой файл
- Все пути к файлам обрабатываются кроссплатформенно
- База данных создаётся автоматически при первом запуске
- Изображения копируются в папку `images/<тип испытания>/` при добавлении
